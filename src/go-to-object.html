Go to object

Input examples:

tickets
Go to tickets list
ticket 123
Go to ticket with id = 123
Ticket#2021031415926535
Go to ticket with number = 2021031415926535
issues
Go to issues list
issue 123
Go to issue 123
blocks
Go to blocks list
block 123
Go to block 123
redactions
Go to redactions list
redaction 123
Go to redaction 123
changeset #12345 or changeset 12345 or cset 12345 or c12345
Go to changeset 12345
changeset #12345 xml
Go to xml representation of changeset 12345
node #123456 or node 123456 or node/123456 or n123456
Go to node 123456
node #123456 version 7 or node #123456v7 or node 123456v7 or node/123456v7 or node/123456/7 or n123456v7
Go to node 123456 version 7
node #123456 xml or node #123456.xml
Go to xml representation of node 123456
node #123456 json
Go to json representation of node 123456
way #23456 ...
relation #3456 ...
Similarly go to other element types
user username
Go to username's profile
uid #312
Go to xml representation of user with id = 312 because there's no user page accessible by id in OSM web UI
